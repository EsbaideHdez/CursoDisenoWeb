Repositorios y commit

Existen dos maneras de crear un repositorio. La primera es desde un path de tu computadora (local) y la segunda importa el repositorio ya existente que se aloja en internet (remoto)

git init

Inicializa un repositorio en un directorio existente. Local

Los archivos estarán sin seguimiento (untracked). Por lo que habrá que usar el comando git add

git clone [url]

Clona/importa un repositorio existente en otro servidor. Remoto

Los archivos estarán bajo seguimiento (tracked) hasta que se haga alguna modificación o se agregue un nuevo archivo

git status

Te permite ver el estado actual del repositorio

git add [archivo/directorio]

Inicializa el seguimiento (tracked) del archivo/directorio especificado.

Agrega el archivo/directorio al staging index, que es la fase anterior para crear una nueva versión. Para crear la versión hace falta confirmación con git commit

git add .

Si se tienen varios archvios o directorios en el repositorio, para no tener que agregar archivo por archivo se usa el punto y se agregarán todos los archvios y directorios al staging index

git rm --cached [archivo]

Elimina un archivo en seguimiento (tracked). Remueve el archivo del staging index

git commit -m "notaDeLaNuevaVersión"

Confirma los cambios agregados con git add y crea una nueva versión en el repositorio

git push [remoto][rama]

Envía los cambios al repositorio remoto (internet)

git pull [identificador del repositorio remoto] [rama a la que se importarán los cambios]

Ejemplo: git pull Origin master

Si se hicieron modificaciones en el repositorio remoto, se usa este comando para importar los cambios al repositorio local

git remote

Te permite ver cuáles son los repositorios remotos vinculados. Si se clonó un repositorio remoto y no se nombró, git por defecto lo nombra Origin

git remote -v

Muestra el nombre del repositorio y su dirección URL

git log

Muestra los commit's que se han creado

git log --stat

Muestra los commit's que se han creado, más un resumen de las estadísticas (cambios hechos). Por ejemplo, cuántos y cuales archvios se modificaron, cuántas líneas de código se agregaron y cuántas se eliminaron

git log --oneline

Muestra los commit's en una sola línea. Solo muestra el identificador del commit y su comentario

git log --graph

Descripción

git log --online --graph --all

Descripción

git diff

Muestra la diferencia entre el commit pasado y el contendio modificado, es decir, muestra el código que se creó después del último commit en letras verdes

git diff [identificador del commit] [archivo]

Ejemplo: git diff f3be550 index.html

Muestra las modificaciones de los commit's (versiones) especificadas. En letras verdes lo que se agregó y en letras rojas lo que se eliminó

git checkout [identidicador del commit]

Ejemplo: git checkout f3be550

checkout se usa para varios casos. En el caso de los commit's, se usa para regresar en el tiempo de un commit (versión) a otra. Se debe tener mucho cuidado porque si se regresa a un commit anterior, todos los que se hayan creado después desaparecerán


NOTA: Al hacer un checkout y regresar a un commit del pasado, se crea una rama por default (HEAD) y al poner git branch apunta a esa rama. *Si queremos regresar al estado anterior de regresar a un commit del pasado, hay que hacer un checkout master para regresar a la rama principal y la rama head desaparecerá*. Si en realidad si se desea regresar al commit del pasado y perder los commit futuros a ese, hay que hacer un merge para pasar los cambios de la rama head a master

Ramas (Branches)

Una rama es una versión del código principal. Su función es crear una copia del código sin afectar al código original (en la rama master) y si los cambios o la adición de nuevo código se quiere agregar al código principal (master) se hace un merge. Algo interesante es que se pueden crear archivos en las nuevas ramas y no se crean en la rama master

git branch [nombre de la rama]

Crea una nueva rama

git checkout [nombre de la rama]

Cambia de rama sin afectar la rama en la que se estaba antes

git checkout -b [nombre de la rama]

Es un atajo de los dos ejemplos anteriores. Crea una nueva rama y se cambia a dicha rama

git branch

Muestra las ramas creadas y en cual de estas se está

git branch -d [nombre de la rama]

Elimina la rama especificada. Para poder eliminar la rama no se debe estar situado en ella, se borra desde otra rama

git merge [nombre de la rama]

Actualiza los cambios de otras ramas a la rama master. Este comando se hace desde la rama master

Subiendo repositorio a Github desde un repositorio local previamente creado

Si se creó un repositorio local con git init y se quiere subir a un repositorio remoto, hay que ir a Github y crear un nuevo repositorio. Al crearlo hay que usar este comando en Git Bash en el path del repositorio local:

Nota: Es el primer comando: git remote add ...
Origin es el nombre por defeto, pero se puede cambiar de nombre. Este nombre es con el que se identifica al repositorio

git remote add [identificador del repositorio remoto] [URL del repositorio remoto]

Ejemplo: git remote add DisenoWeb https://github.com/renebarrera1022/CursoDisenoWeb.git

Este comando se usa desde un repositorio local ya creado, lo que hace es vincular el repositorio local con el remoto

git push -u [identificador del repositorio remoto] [nombre de la rama desde donde se subirá]

-u: guarda (remoto)(rama), por lo que la siguiente vez que desee subir cambios solo hay que poner: git push

Ejemplo: git push -u DisenoWeb master

Sube los archivos de la rama especificada al repositorio remoto especificado
